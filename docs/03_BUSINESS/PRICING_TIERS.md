## 1) Commercial Model Overview

GovFlow is offered in three commercial tiers:
- **Minimal**
- **Standard** (recommended default for UAE public sector)
- **Enterprise**

And three licensing/pricing structures:
1. **Per User Per Month (PUPM)**
2. **Enterprise License (Unlimited Users)**
3. **On-Prem License (One-Time + Annual Support)**

This allows procurement teams to choose the right model by budget cycle, policy requirements, and expected scale.

---

## 2) Tier Definitions (Service Packaging)

| Tier | Target Customer | Deployment Model | Infrastructure Type |
|---|---|---|---|
| **Minimal** | Department pilots, smaller entities | Full Cloud (default) | Managed cloud baseline |
| **Standard** | Ministries, authorities, regulated organizations | **Hybrid (recommended default)** | Cloud + On-Prem integration |
| **Enterprise** | Sovereign or mission-critical national programs | Hybrid dedicated or On-Prem/Private Cloud | Dedicated sovereign-grade architecture |

### Minimal
- **Features included:** core workflows, approvals, RBAC, baseline dashboards, standard branding
- **SLA level:** 99.5%+ availability target, 15-min critical response, 4-hour critical resolution
- **Support model:** shared support, standard escalation, periodic service review

### Standard
- **Features included:** all Minimal + advanced workflows, broader integrations, stronger governance controls, expanded analytics
- **SLA level:** 99.5%+ availability target, 15-min critical response, 4-hour critical resolution
- **Support model:** priority queue, named governance contacts, structured optimization reviews

### Enterprise
- **Features included:** all Standard + dedicated environment options, sovereign control profile, advanced continuity/governance package
- **SLA level:** 99.5%+ availability target with mission-critical operating controls
- **Support model:** 24/7 mission-critical support posture, senior governance escalation

## 3) Pricing Model Options

## 3.1 Per User Per Month (PUPM)
**Best for:** predictable growth organizations and phased rollouts

### Commercial structure
- Monthly or annual billing based on active licensed users
- Tier-based feature bundles (Minimal / Standard / Enterprise)
- Optional volume discounts by user bands

### Typical inclusions
- Platform access and standard feature set by tier
- Standard updates and maintenance
- Baseline support package

### Typical exclusions (priced separately)
- Major custom development
- Non-standard integrations
- Dedicated environment isolation (if required)

---

## 3.2 Enterprise License (Unlimited Users)
**Best for:** ministries, large authorities, and group entities with broad user bases

### Commercial structure
- Fixed annual platform license for defined organization scope
- Unlimited named users within contracted legal entity scope
- Multi-entity expansion terms can be added in contract schedule

### Typical inclusions
- Full platform usage under selected tier
- Governance and service review framework
- Standard or premium support (based on plan)

### Typical exclusions (priced separately)
- Specialized sovereign enhancements outside agreed scope
- New major modules not in signed scope
- Custom integration factory services

---

## 3.3 On-Prem License (One-Time + Annual Support)
**Best for:** sovereign, high-security, or policy-mandated on-prem deployments

### Commercial structure
- One-time software license fee
- Mandatory annual support and maintenance fee (typically percentage of license)
- Optional professional services for implementation and upgrades

### Typical inclusions
- Licensed software deployment rights per contract scope
- Security patches, maintenance releases, and support coverage
- Version upgrade rights according to support agreement

### Typical exclusions (priced separately)
- Client-owned infrastructure/hardware costs
- Datacenter operations and third-party tooling costs
- Customizations beyond baseline contract scope

## 4) What Is Included vs. Extra

| Component | Included (Base) | Extra / Optional |
|---|---|---|
| Platform software | Yes (all models) | N/A |
| Hosting | Included in cloud-managed packages | On-Prem hosting always client-side |
| Support | Included per SLA tier | Premium SLA uplift |
| White-label branding | Included (standard branding scope) | Advanced UX/custom branding packages |
| Integrations | Baseline connectors | Advanced/custom enterprise integrations |
| Customization | Minor configuration by tier | Custom modules, bespoke workflows, major feature extensions |

## 5) Government Procurement Model (UAE Contracting)

Recommended contracting approach for UAE public sector:

- **Primary structure:** annual contract with renewal options
- **Preferred term:** 3-year framework (1+1+1 or 2+1) for budget and continuity alignment
- **Commercial mechanism:** annual invoice cycle with defined service milestones
- **Governance:** quarterly service review + annual commercial/technical review

### Multi-year commercial guidance
- Year 1: full implementation + onboarding + stabilization
- Year 2 onward: recurring platform and support services
- Multi-year discount can be applied for committed contract duration and volume

## 6) Optional Add-Ons

## 6.1 AI Module Add-On
- AI-assisted workflow insights, classification, and decision support capabilities
- Packaged as optional licensed module
- Requires separate scope, governance, and policy approvals

## 6.2 Integration Pack Add-On
- ERP/HR/CRM/document systems integration bundles
- Option for API gateway and integration management services
- Can be procured per connector or as enterprise integration package

## 6.3 Premium SLA Add-On
- Enhanced service governance model
- Increased review cadence and dedicated service management
- Mission-critical operational options for sensitive programs

---

## 7) Procurement Positioning Guidance

For UAE procurement proposals:
- Position **Standard tier + Hybrid deployment** as default recommendation.
- Position **Enterprise + On-Prem/Private Cloud** for strict sovereign requirements.
- Position **Minimal** for pilot and phased departmental adoption.

This gives committees a clear path from initial deployment to national-scale expansion without platform change.

---

## 8) Commercial Notes for RFP Submission

- Final pricing values should be issued in the commercial schedule based on:
  - user volume or organization scope
  - deployment model (cloud/hybrid/on-prem)
  - integration complexity
  - customization and SLA uplift level
- Proposal should include:
  - inclusions/exclusions matrix
  - implementation milestones
  - support and escalation model
  - renewal and change-control terms
